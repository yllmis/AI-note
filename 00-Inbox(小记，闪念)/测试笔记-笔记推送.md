# Redis 缓存穿透与布隆过滤器

## 缓存穿透

缓存穿透是指查询一个不存在的数据，缓存和数据库都没有，导致请求直接打到数据库。

**解决方案**：
1. 缓存空对象 — 即使查不到也缓存一个空值，设置短过期时间
2. 布隆过滤器 — 在缓存前加一层布隆过滤器，判断 key 是否存在

## 布隆过滤器原理

布隆过滤器是一个 bit 数组 + 多个哈希函数。插入时对 key 做多次哈希，将对应位设为 1；查询时如果任意一位为 0 则一定不存在。

**特点**：判断"不存在"是确定的，判断"存在"可能有误判。

## Redis 缓存击穿

缓存击穿是指一个热点 key 过期，大量并发请求同时打到数据库。

**解决方案**：
1. 互斥锁（SETNX）
2. 逻辑过期（不设物理过期时间，后台异步刷新）

## Redis 缓存雪崩

缓存雪崩是指大量缓存同时过期，导致请求全部打到数据库。

**解决方案**：
1. 过期时间加随机值
2. 缓存预热 + 多级缓存
3. 限流降级
