# 笔记推送系统 — V1 实现方案

- **开发目录**：`/Users/yllmis/go_projects/note_feed/`
- **Go 模块**：`note_feed`

## 项目结构

```
note_feed/
├── cmd/
│   └── note_feed/
│       └── main.go           # CLI 入口，cobra 命令注册
├── internal/
│   ├── config/
│   │   └── config.go          # 配置读取（yaml + 环境变量覆盖）
│   ├── db/
│   │   ├── sqlite.go          # SQLite 初始化 + 自动建表
│   │   └── models.go          # 数据操作（pending_diffs / pushed_articles / push_logs）
│   ├── git/
│   │   └── git.go             # git diff/log 操作封装
│   ├── llm/
│   │   └── deepseek.go        # DeepSeek API 调用（提炼 + 摘要 + 补充生成）
│   ├── search/
│   │   ├── juejin.go          # 掘金搜索源
│   │   ├── google.go          # Google Custom Search 兜底
│   │   └── search.go          # 搜索编排（多源 + 去重 + 降级）
│   └── push/
│       ├── feishu.go          # 飞书 Bot Webhook 推送
│       ├── mail.go            # SMTP 邮件推送
│       └── push.go            # 推送编排（格式组装 + 多路发送）
├── config.yaml                # 配置文件模板
├── go.mod
└── go.mod
```

> [!tip] 为什么用 cobra
> Go CLI 生态事实标准，子命令路由（`cache-diff` / `push` / `init`）天然支持，不需要自己解析 os.Args。

## V1 范围（最小可行版本）

| 模块 | V1 做 | V1 不做 |
|------|-------|---------|
| Git 捕获 | post-commit hook + `cache-diff` 子命令 | git log 兜底（已在设计中预留位置，暂不实现） |
| LLM 提炼 | 调用 DeepSeek API 输出 JSON | prompt 调优、输出校验重试、fallback 模型 |
| 搜索 | **仅掘金 API** | Google Custom Search、LLM 补充生成、embedding 重排序 |
| 推送 | **仅邮件** | 飞书、微信、QQ |
| 配置 | yaml 文件 | 环境变量覆盖、配置热加载 |
| 安装 | 手动 go build | brew tap、自动更新 |

> [!important] V1 原则：先跑通核心链路
> LLM 提炼（最不确定） + 搜索（核心价值）+ 邮件推送（最简单），三个环节串起来能用。hook 安装、配置管理、搜索降级这些做最小可用，不花时间打磨。

## 数据流（V1 简化版）

```
git commit
    │
    ▼
post-commit hook
    │  git diff HEAD~1 HEAD -- '*.md'
    │  → 笔记推送 cache-diff
    ▼
SQLite pending_diffs ←──────────────────┐
    │                                    │
    ▼ (每日 20:00 cron / 手动执行)         │ 兜底补丁（V1 预留）
笔记推送 push --daily                     │
    │ 从 pending_diffs 聚合当日 diffs       │ 若 pending_diffs 为空
    ▼                                    │ 从 git log 补拉
DeepSeek V4 提炼知识点                     │
    │ 输入：diff 文本                      │
    │ 输出：{topics: [{category, keywords, extensions}]}
    ▼
关键词拼接 → 掘金 API 搜索
    │ 每个大类取前 2 关键词 + 中文分类名
    ▼
去重（SQLite pushed_articles URL 精确匹配）
    │ 每个大类最多 3 篇
    ▼
DeepSeek V4 生成摘要
    │ 每篇文章调一次，生成 1-2 句摘要
    ▼
组装邮件 HTML → SMTP 发送
```

## 配置设计

```yaml
# config.yaml
vault_path: ""  # 留空则用 os.Getwd()

llm:
  provider: deepseek
  api_key: "${DEEPSEEK_API_KEY}"  # 支持环境变量引用
  model: deepseek-v4
  timeout: 30s

search:
  juejin:
    enabled: true
    # 掘金搜索不需要 API key
  google:
    enabled: false  # V1 不开
    api_key: ""
    cse_id: ""

push:
  email:
    smtp_host: smtp.qq.com
    smtp_port: 587
    username: "${EMAIL_USER}"
    password: "${EMAIL_PASS}"
    to: "${EMAIL_TO}"
  feishu:
    enabled: false  # V1 不开
    webhook_url: ""

db:
  path: ./data/push.db  # vault 相对路径
```

> [!note] key 安全
> API key 不写在 yaml 里，yaml 中写 `${VAR_NAME}` 占位，运行时从环境变量读取。支持完整 `os.ExpandEnv` 替换。

## CLI 命令设计

```
笔记推送 init
  - 在当前目录写入 config.yaml（如不存在）
  - 安装 post-commit hook 到 .git/hooks/
  - 初始化 SQLite 数据库（建表）

笔记推送 cache-diff --commit-hash=<hash>
  - 从 stdin 读取 diff 内容
  - 写入 pending_diffs 表
  - exit code 0 表示成功，不阻塞 commit

笔记推送 push --daily [--date 2026-05-15]
  - 默认处理今日 pending diffs
  - --date 指定日期用于回溯/测试
  - 完整走完：聚合 → LLM 提炼 → 搜索 → 摘要 → 推送
  - 结果写入 push_logs

笔记推送 push --test
  - 发送测试邮件验证配置正确
```

## 关键设计决策

### 1. SQLite 驱动：modernc.org/sqlite（纯 Go，无 CGO）

不用 `go-sqlite3`（需要 gcc + CGO），modernc 是 CGo-free 的纯 Go 实现，`go build` 无需任何本地依赖，与"单二进制"目标完全吻合。

### 2. LLM 调用：直接 HTTP（不依赖 SDK）

DeepSeek 兼容 OpenAI API 格式，直接用 `net/http` + 标准请求体即可。不需要引入 openai Go SDK（依赖重且版本耦合）。

```go
// internal/llm/deepseek.go
type Message struct {
    Role    string `json:"role"`
    Content string `json:"content"`
}

type ChatRequest struct {
    Model    string    `json:"model"`
    Messages []Message `json:"messages"`
}

type ChatResponse struct {
    Choices []struct {
        Message Message `json:"message"`
    } `json:"choices"`
}
```

### 3. 邮件推送：直接 net/smtp

Go 标准库 `net/smtp` 支持 STARTTLS（端口 587），无需第三方库。QQ 邮箱 / Gmail 都兼容。

```go
func SendEmail(cfg SMTPConfig, subject, htmlBody string) error {
    // 1. 连接 smtp_host:587
    // 2. STARTTLS 升级
    // 3. Auth LOGIN
    // 4. 发送 html 邮件
}
```

### 4. diff 输入范围限制

post-commit hook 中 `-- '*.md'` 过滤非笔记文件变更，避免图片/附件等二进制内容被送入 LLM、浪费 token。

> diff 大小超过 20KB 时，`cache-diff` 自动截断告警（`[truncated ...]`），前端提炼 prompt 加说明"部分 diff 因过长被截断"。

### 5. 掘金搜索实现

掘金开放搜索接口（无需 token）：

```
GET https://api.juejin.cn/search_api/v1/search?keyWord={关键词}&page=0&pageSize=10
```

V1 直接用 HTTP GET，返回 JSON 解析 title / url / description。
