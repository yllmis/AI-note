# 虚拟化
## 调度
### 进程调度
FIFO（先进先出） -> SJF（最短任务优先） -> STCF（最短完成时间优先）

### 多级反馈队列
MLFQ

+ 规则一：A的优先级 > B的优先级，运行A（不运行B）
+ 规则二：A的优先级 = B的优先级，轮转运行
+ 规则三：工作进入系统时，放入最高优先级
+ 规则四：工作用完了在某一层中的时间配额（无论中间主动放弃多少次），就降低优先级
+ 规则五：经过一段时间S，就将系统中的所有工作重新加入最高优先级队列

### 比例份额
彩票机制（没有最优的分配方案）



步长调度 每个工作有自己的步长，与票值成反比。

票值： A 100、B 50、C 250  （通过一个大数除以票值，比如 10000）

步长： A 100、B 200、C 40

刚开始：所有进程都可能被选择执行。

+ 先A，A执行一个时间片，步长变为100
+ 再B，执行一个时间片，步长变为200
+ 再C，执行一个时间片，步长变为40
    - 这时算法选择最短行程，于是继续执行C，步长变为80

## 锁
### PETERSON算法（纯软件锁） 仅适用于两个线程
```cpp fold title:PETERSON算法
int flag[2];
int turn;

void init() {
	flag[0] =falg[1] = 0;
	turn = 0;
}
void lock() {
	flag[self] = 1;
	turn = 1 - self;
	while((flag[1-self] == 1) && (turn == 1-self))
		;// spin-wait
}
void unlock(){
	flag[self] = 0;
}
```
我们把 Peterson 算法看作是两个非常有礼貌的室友（线程 0 和线程 1）在争用同一个卫生间。理解了这个比喻，代码的逻辑就全通了：
#### 1. 变量的精准定义

- **`flag` 数组（意愿声明）：**
    
    - `flag[self] = 1` 意思是：“**我现在想用卫生间了！**”
    - 这并不代表它已经进去了，只是举手表达了强烈的意愿。
- **`turn` 变量（谦让标志）：**
    
    - `turn = 1 - self` 意思是：“**如果你也刚好想用，那你先请。**”
    - 这是一个单值变量，无论是谁去写，它最终只能记录一个值。它完美解决了“两人同时走到门口，僵持不下”的死锁问题。

#### 2. 逐行拆解 `lock()` 函数的魔法

当一个线程（假设是线程 0，`self = 0`）调用 `lock()` 时，发生了这三步：

1. **`flag[0] = 1;`** （宣告意愿：我想进去！）
2. **`turn = 1;`** （表现风度：如果线程 1 也想进，让线程 1 先进！）
3. **`while ((flag[1] == 1) && (turn == 1)) ;`** （这是算法的灵魂：**我在门口死等（自旋）的条件是什么？**）

这个 `while` 循环的等待条件必须**同时满足两个要求**才会死等：

- `flag[1] == 1`：对方也想进去。
- `turn == 1`：而且现在刚好是我让给对方的轮次。

只要打破其中**任何一个条件**，线程 0 就可以大摇大摆地进入卫生间：

- **打破条件 A：** 对方根本不想进（`flag[1] == 0`）。那我不管 `turn` 是谁，我直接进！
- **打破条件 B：** 对方也想进（`flag[1] == 1`），但是对方比我晚一步表现了风度，把 `turn` 改成了 0 （`turn == 0`）。这意味着对方对我说“你先请”，那我就不客气了，直接进！

#### 3. 三种核心场景推演

为了彻底搞懂，我们推演一下这两种常见情况：

**场景一：毫无竞争（线程 1 在睡觉，只有线程 0 想进）**

- 线程 0 执行：`flag[0] = 1`，`turn = 1`。
- 走到 `while` 检查：此时 `flag[1]` 是 0。
- 结果：`while` 条件不成立（`flag[1] == 1` 为假），线程 0 瞬间通过，进入临界区。

**场景二：激烈冲突（线程 0 和线程 1 同时调用 `lock`）** 这就是 Peterson 算法最天才的地方。假设他们完全同时执行：

1. 两人同时举手：`flag[0] = 1`，`flag[1] = 1`。
2. 两人互相谦让：线程 0 执行 `turn = 1`，线程 1 执行 `turn = 0`。
3. **重点来了：** 内存里的 `turn` 只能保存一个值。谁后执行，`turn` 就是谁赋的值。假设线程 1 稍微慢了万分之一秒，`turn` 最终变成了 `0`。
4. 现在两人同时判断 `while`：
    - **线程 0 看 `while`：** 对方想进（`flag[1]==1`），但此时 `turn == 0`（不是 1）。条件为假，**线程 0 成功进入！**
    - **线程 1 看 `while`：** 对方想进（`flag[0]==1`），且此时 `turn == 0`（等于 1-self）。条件为真，**线程 1 被卡在 `while` 循环里原地打转（自旋等待）**。

**场景三：释放锁 `unlock()`** 当线程 0 用完卫生间，调用 `unlock()` 时，它只做了一件事：

- `flag[0] = 0;` （我不用了，撤销意愿）。
- 此时，原本在门外死等的线程 1，它的 `while` 循环里 `flag[0] == 1` 瞬间变成了假，循环被打破，线程 1 终于可以进去了。

# 内存
## 虚拟内存

### 介绍
虚拟内存是相对于物理内存来说的，物理内存，简单来说就是电脑上看到的那些内存条，是机器真正可以实际访问的物理内存空间。比如内存条是 8G 的，那计算机可用的物理内存就是 8G。

为了防止多进程运行时造成的这些使用内存地址的冲突，操作系统引入了虚拟内存，为每个进程提供了一个独立的虚拟内存空间，使得进程以为自己独占全部内存资源。
![[Pasted image 20260516103349.png]]
由于每个进程都有一个这么大的地址空间，导致所有进程的虚拟内存加起来，自然要比实际物理内存大得多。所以，并不是所有的虚拟内存都会分配物理内存，只有那些实际使用的虚拟内存才会分配物理内存。

当进程对某块虚拟内存进行读写时，CPU 就会去访问这块内存， 这时如果发现这块虚拟内存没有映射到物理内存， CPU 就会产生缺页中断，进程会从用户态切换到内核态，并将缺页中断交给内核的缺页中断函数处理，这时才会真正地为它分配物理内存。  
![[Pasted image 20260516104209.png]]

#### 分段
内存分段机制，简单理解就是根据程序申请使用内存的需要，来把物理内存分成一段一段内存来管理，比如程序需要100M的内存，分段机制就给1段100M连续空间的物理内存与之对应。
![[Pasted image 20260516104340.png]]
#### 分页
内存分页将整个虚拟内存和物理内存空间分成一段段固定大小的片，虚拟内存和物理内存的映射以这个片为最小单位进行管理，我们把这个片称为页，在Linux系统上，页的大小为4KB。
![[Pasted image 20260516104357.png]]
此外，为了解决页表过大的问题，操作系统引入了多级页表机制。
![[Pasted image 20260516104701.png]]

为了解决页表访问慢的问题，在CPU中加入了TLB页表缓存机制。
![[Pasted image 20260516104707.png]]

### 为什么有虚拟内存 
虚拟内存是对物理内存的抽象，每个进程看到独立地址空间，由系统负责映射到物理内存/磁盘。面试里可按三点展开：进程隔离提升安全性；支持按需调页，扩大可用内存；降低对连续物理内存的要求。
### 内存分页的好处和问题
分页把内存按固定大小管理，提升利用率与管理效率，但会带来页表和缺页开销。面试里可按两点展开：优势：减少外部碎片、支持虚拟内存；问题：内部碎片、TLB miss 和缺页成本。
