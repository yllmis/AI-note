# 他人面经

## 来源：2023 牛客

### Golang

#### Q1：map 的 key 有序吗？

**答案**：
无序。Go 的 map 底层是哈希表，key 通过哈希函数分散存储，遍历顺序由哈希值决定。Go 运行时还会**故意随机化遍历起点**，防止开发者依赖遍历顺序。

底层实现：
- Go 1.23 及之前：`hmap` 桶模型（拉链法，每个桶 8 个 kv）
- Go 1.24+：**Swiss Table**（开放寻址法，内存局部性更好，性能更高）

如果需要有序遍历，先把 key 取出来排序，再按排序后的 key 读 map。

**记忆**：**map key 无序，哈希决定位置，运行时随机起点防依赖；1.24 换 Swiss Table 开放寻址**。

#### Q2：slice 深度拷贝



**答案**：
slice 是引用类型，直接赋值是浅拷贝（共享底层数组）。深度拷贝需要**分配新数组 + 复制数据**：

```go
// 方法1：copy
dst := make([]int, len(src))
copy(dst, src)

// 方法2：append
dst := append([]int(nil), src...)
```

注意：如果 slice 元素是指针或包含指针（如 `[]*int`、`[]struct`），以上只拷贝了指针值，元素指向的对象仍是同一个。要真正深拷贝需要逐个元素递归复制。

**记忆**：**slice 赋值浅拷贝共享底层数组；深拷贝用 copy 或 append；元素含指针需递归复制**。

#### Q3：interface{} 怎么用

**答案**：
`interface{}`（Go 1.18+ 也可写作 `any`）是空接口，所有类型都实现了它，可以接收任意类型的值。

```go
// 接收任意类型
func Print(v interface{}) {
    fmt.Println(v)
}

// 类型断言取出具体值
func Process(v interface{}) {
    switch val := v.(type) {
    case int:
        fmt.Println("int:", val)
    case string:
        fmt.Println("string:", val)
    default:
        fmt.Println("unknown:", val)
    }
}
```

使用场景：通用函数、JSON 反序列化、容器存储不同类型。Go 1.18+ 优先用泛型替代 `interface{}`。

**记忆**：**interface{} 是空接口，任何类型都能赋值；类型断言或 type switch 取出具体值；泛型时代优先用 any 约束**。

#### Q4：说一下 Go 的继承

**答案**：
Go **没有继承**，用**组合**实现代码复用。

```go
type Animal struct {
    Name string
}

func (a Animal) Speak() {
    fmt.Println(a.Name, "speaks")
}

type Dog struct {
    Animal // 嵌入（组合），不是继承
    Breed  string
}

d := Dog{Animal: Animal{Name: "Buddy"}, Breed: "Labrador"}
d.Speak() // 可以直接调用 Animal 的方法（方法提升）
```

和继承的区别：
- Go 是**方法提升**（method promotion），不是真正的继承
- `Dog` 不是 `Animal` 的子类型，不能赋值给 `Animal` 类型的变量（但可以赋值给 `Animal` 字段）
- 优先用组合，不要刻意模拟继承

**记忆**：**Go 没有继承用组合；嵌入结构体方法自动提升；不是子类型，不能当父类型赋值**。

### 数据结构

#### Q5：二叉树遍历方式

**答案**：
四种基本遍历：

| 遍历方式 | 顺序 | 用途 |
|---------|------|------|
| 前序（Pre-order） | 根→左→右 | 复制树、前缀表达式 |
| 中序（In-order） | 左→根→右 | BST 有序输出 |
| 后序（Post-order） | 左→右→根 | 删除树、后缀表达式 |
| 层序（Level-order） | 逐层从左到右 | BFS、最短路径 |

前三种可用递归或栈（迭代），层序用队列（BFS）。

**记忆**：**前根中根后根，层序用队列；中序遍历 BST 得有序序列**。

#### Q6：深度优先搜索（DFS）和广度优先搜索（BFS）

**答案**：

**DFS（深度优先）**：
- 沿一条路径走到底，再回溯
- 实现：递归 或 栈
- 空间复杂度 O(h)，h 为树高
- 适用：路径搜索、连通性判断、回溯问题

**BFS（广度优先）**：
- 逐层扩展，先访问所有邻居
- 实现：队列
- 空间复杂度 O(w)，w 为最大宽度
- 适用：最短路径、层序遍历、拓扑排序

**记忆**：**DFS 用栈/递归走到底再回溯；BFS 用队列逐层扩展；DFS 适合路径搜索，BFS 适合最短路径**。

### 操作系统

#### Q7：原子操作（多线程执行 i++ 时怎么发生冲突）

**答案**：
`i++` 不是原子操作，实际包含三步：
1. **读**（Load）：从内存读 i 的值到寄存器
2. **改**（Add）：寄存器中 i+1
3. **写**（Store）：写回内存

多线程下可能交错执行：
```
线程A: Load i=0
线程B: Load i=0        // A 还没写回，B 读到旧值
线程A: Add → 1, Store i=1
线程B: Add → 1, Store i=1  // 期望是 2，实际是 1
```

解决方案：
- **原子指令**：CPU 提供 CAS（Compare-And-Swap）、LOCK 前缀指令，保证读改写不可分割
- **互斥锁**：同一时刻只有一个线程执行 i++
- Go 中用 `atomic.AddInt64(&i, 1)` 或 `sync.Mutex`

**记忆**：**i++ 是读改写三步，多线程会交错导致丢失更新；用原子操作或锁保证不可分割**。

#### Q8：内核态和用户态的区别

**答案**：

| | 用户态 | 内核态 |
|---|--------|--------|
| 权限 | 受限，不能直接访问硬件和内核数据 | 最高权限，可访问所有资源 |
| 运行内容 | 应用程序 | 操作系统内核、驱动 |
| 地址空间 | 用户空间（低地址） | 内核空间（高地址，独立映射） |
| 切换方式 | 系统调用、中断、异常 | - |

切换过程：用户态 → **保存现场** → 切换到内核栈 → 执行内核代码 → **恢复现场** → 返回用户态。

为什么要区分：保护内核安全，防止应用程序误操作破坏系统。

**记忆**：**用户态权限受限，内核态最高权限；通过系统调用切换；保护内核不被应用程序破坏**。

#### Q9：介绍一下 epoll

**答案**：
epoll 是 Linux 下高性能的 IO 多路复用机制，用于同时监听大量文件描述符（fd）的 IO 事件。

和 select/poll 的区别：
- **select**：fd 数组拷贝到内核、遍历检查、fd 上限 1024
- **poll**：链表存储无上限，但仍需遍历
- **epoll**：红黑树管理 fd + 就绪链表，**只返回就绪的 fd**，O(1) 获取结果

epoll 三个核心 API：
1. `epoll_create`：创建 epoll 实例
2. `epoll_ctl(ADD/MOD/DEL)`：增删改监听的 fd
3. `epoll_wait`：阻塞等待就绪事件

**适用场景**：大量连接但活跃连接少（如 Web 服务器、长连接服务）。

**记忆**：**epoll 红黑树+就绪链表，只返回就绪 fd；select/poll 需遍历；大量连接少量活跃用 epoll**。

#### Q10：epoll 的边沿触发和水平触发

**答案**：

**水平触发（LT，Level Triggered，默认）**：
- 只要 fd 的缓冲区有数据可读，每次 `epoll_wait` 都会返回该 fd
- 编程简单，数据没读完下次还会通知
- 缺点：可能重复通知

**边沿触发（ET，Edge Triggered）**：
- 只在 fd 状态**变化时**通知一次（从未就绪→就绪）
- 必须一次读完所有数据（循环读到 EAGAIN），否则不会再通知
- 必须配合**非阻塞 IO**
- 性能更好，减少 epoll_wait 调用次数

```go
// ET 模式必须循环读
for {
    n, err := conn.Read(buf)
    if err != nil {
        if err == syscall.EAGAIN {
            break // 读完了
        }
        break
    }
    process(buf[:n])
}
```

**记忆**：**LT 默认有数据就通知，编程简单；ET 只通知一次状态变化，必须读完+非阻塞；ET 性能好但编程复杂**。

### 计算机网络

#### Q11：TCP 和 UDP 的区别

**答案**：

| | TCP | UDP |
|---|-----|-----|
| 连接 | 面向连接（三次握手） | 无连接 |
| 可靠性 | 可靠（确认、重传、排序） | 不可靠 |
| 传输方式 | 字节流 | 数据报 |
| 速度 | 较慢（开销大） | 较快（开销小） |
| 一对多 | 一对一 | 一对多、多对多 |
| 头部大小 | 20 字节 | 8 字节 |
| 流控/拥塞 | 有滑动窗口、拥塞控制 | 无 |

典型应用：TCP → HTTP/HTTPS、FTP、SMTP；UDP → DNS、视频直播、游戏。

**记忆**：**TCP 可靠有序连接，UDP 快但不可靠；TCP 一对一，UDP 一对多；TCP 有流控拥塞控制**。

---

#### Q12：UDP 有发送长度限制吗？

**答案**：
有。UDP 数据报最大 **65535 字节**（IP 头部 20 字节 + UDP 头部 8 字节 = 最大 payload 65507 字节）。

实际限制：
- **理论最大**：65535 字节（IP 包最大长度）
- **实际建议**：< 1472 字节（避免 IP 分片，MTU 通常 1500）
- 超过 MTU 会触发 IP 分片，增加丢包风险

如果数据大于 65507 字节，需要应用层分片。

**记忆**：**UDP 最大 65535 字节，实际建议 <1472 避免分片；超过 MTU 会 IP 分片**。

### 编程题

#### Q14：数组第 K 大元素（口述）

**答案**：

**方法1：排序** — O(n log n)
```go
sort.Ints(nums)
return nums[len(nums)-k]
```

**方法2：快速选择（QuickSelect）** — 平均 O(n)，最差 O(n²)
类似快排的 partition，每次确定一个 pivot 的最终位置：
- pivot 位置 == len-k → 找到
- pivot 位置 > len-k → 在左边找
- pivot 位置 < len-k → 在右边找

**方法3：最小堆** — O(n log k)
维护大小为 k 的最小堆，遍历数组，堆顶就是第 k 大。

面试最优解：快速选择（平均 O(n)）或最小堆（O(n log k)）。

**记忆**：**排序 O(n log n) 最简单；QuickSelect 平均 O(n) 最优；最小堆 O(n log k) 适合流数据**。

#### Q15：合并两个有序递增数组

**答案**：

**双指针法** — O(n+m)：

```go
func merge(a, b []int) []int {
    result := make([]int, 0, len(a)+len(b))
    i, j := 0, 0
    for i < len(a) && j < len(b) {
        if a[i] <= b[j] {
            result = append(result, a[i])
            i++
        } else {
            result = append(result, b[j])
            j++
        }
    }
    result = append(result, a[i:]...)
    result = append(result, b[j:]...)
    return result
}
```

从后往前合并（如果目标数组空间足够）可以避免额外空间：

```go
// 从后往前，写入 nums1 的尾部
func merge(nums1 []int, m int, nums2 []int, n int) {
    i, j, k := m-1, n-1, m+n-1
    for i >= 0 && j >= 0 {
        if nums1[i] >= nums2[j] {
            nums1[k] = nums1[i]
            i--
        } else {
            nums1[k] = nums2[j]
            j--
        }
        k--
    }
    for j >= 0 {
        nums1[k] = nums2[j]
        j--
        k--
    }
}
```

**记忆**：**双指针从头比较取小的；从后往前合并可省空间；O(n+m) 时间复杂度**。

#### Q16：用两个队列实现栈（口述）

**答案**：

核心思路：**始终保持一个队列为空**，入栈时往非空队列 push，出栈时把非空队列的元素依次转移到空队列，最后剩下的就是栈顶。

```go
type MyStack struct {
    q1, q2 []int
}

func (s *MyStack) Push(x int) {
    // 往非空队列 push
    if len(s.q1) > 0 {
        s.q1 = append(s.q1, x)
    } else {
        s.q2 = append(s.q2, x)
    }
}

func (s *MyStack) Pop() int {
    // 把非空队列的 n-1 个元素转移到空队列
    from, to := &s.q1, &s.q2
    if len(s.q2) > 0 {
        from, to = &s.q2, &s.q1
    }
    for len(*from) > 1 {
        *to = append(*to, (*from)[0])
        *from = (*from)[1:]
    }
    val := (*from)[0]
    *from = (*from)[1:]
    return val
}
```

关键：入栈 O(1)，出栈 O(n)。

**记忆**：**一个队列存数据，出栈时把前 n-1 个转移到另一个队列，剩下的就是栈顶**。
