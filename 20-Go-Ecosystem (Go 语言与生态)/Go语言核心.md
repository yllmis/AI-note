## 1. GMP调度模型
**答案**：
- G：Goroutine，用户态轻量级线程，包含栈、指令、状态。
- M：Machine，内核线程，绑定CPU。
- P：Processor，逻辑处理器，持有G队列，连接M与G。
流程：M绑定P -> P从本地/全局队列取G执行 -> G阻塞时M解绑P -> P找其他M执行。
**记忆**：**G干活、M跑腿、P派活**。

## 2. Channel
**答案**：
- 有缓冲：异步，满了才阻塞。
- 无缓冲：同步，必须收发同时就绪。
- 关闭后只读不写，重复panic。
**记忆**：**有缓冲异步、无缓冲同步、关闭只收不发**。

## 3. Go GC三色标记
**答案**：
白：未扫描；灰：扫描中；黑：扫描完。
流程：栈根节点灰 -> 遍历引用灰 -> 全部变黑结束。
**记忆**：**白没扫、灰在扫、黑扫完**。

## 4. sync包
**答案**：
- Mutex：互斥锁。
- RWMutex：读共享、写互斥。
- Once：只执行一次。
- WaitGroup：等待一组goroutine。
- Pool：对象复用，减轻GC。
**记忆**：**互斥用Mutex、多读用RWMutex**。

## 5. context
**答案**：
用于goroutine取消、超时、传值。
不要放数据库、不要滥用。
**记忆**：**控制生命周期，不存长数据**。
